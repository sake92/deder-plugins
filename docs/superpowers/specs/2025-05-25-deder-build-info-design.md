# Deder BuildInfo Plugin — Design Spec

## Overview

A `deder-build-info` plugin that generates a Scala `object` containing build metadata
constants at compile time. Modeled after Mill's `BuildInfo` plugin and follows the
existing `deder-protobuf` plugin architecture.

## Configuration (Pkl schema)

### `BuildInfoPlugin` extends `P.DederPlugin`
- `id = "build-info"`
- `deps { "ba.sake::deder-build-info:0.1.0-SNAPSHOT" }`
- `config: BuildInfoPluginConfig`

### `BuildInfoPluginConfig`
- `defaults: ModuleDefaults` — repo-wide defaults
- `modules: Mapping<String, ModuleOverride>` — per-module overrides

### `ModuleDefaults` (all have defaults)
| Field | Type | Default |
|-------|------|---------|
| `enabled` | `Boolean` | `true` |
| `packageName` | `String?` | `null` (root package) |
| `objectName` | `String` | `"BuildInfo"` |
| `includeGitHash` | `Boolean` | `false` |
| `includeTimestamp` | `Boolean` | `false` |
| `extra` | `Mapping<String, String>` | `new {}` |

### `ModuleOverride` (all nullable for optional override)
Same fields as `ModuleDefaults` but all nullable. Null = "not overridden, use default".

## Generated Output

A Scala source file placed in the source generator output directory:

```scala
// Generated by deder-build-info. DO NOT EDIT.
package com.example

object BuildInfo {
  val moduleName: String = "my-app"
  val moduleVersion: String = "0.1.0-SNAPSHOT"
  val scalaVersion: String = "3.7.4"       // absent for Java modules
  val javaVersion: String = "21.0.5"
  val gitHead: String = "abc123def"        // only if includeGitHash
  val builtAtMillis: Long = 1711123200000L  // only if includeTimestamp
  val appEnv: String = "production"         // from extra

  def toMap: Map[String, Any] = Map(...)
}
```

### Constants

| Constant | Always? | Source |
|----------|---------|--------|
| `moduleName` | Yes | `ctx.module.id` |
| `moduleVersion` | Yes | Hardcoded `"0.1.0-SNAPSHOT"` (will depend on `versionTask` when `CoreTasksApi` expands) |
| `scalaVersion` | Yes | Module Scala version; **absent** for `JavaModule`/`JavaTestModule` |
| `javaVersion` | Yes | `System.getProperty("java.version")` |
| `gitHead` | Opt-in | `git rev-parse HEAD` run at task time |
| `builtAtMillis` | Opt-in | `System.currentTimeMillis()` at task time |
| `extra` keys | User | `Mapping<String, String>` from config (appended alphabetically) |

## Architecture

### Directory layout

```
deder-build-info/
  README.md
  resources/
    META-INF/
      services/ba.sake.deder.DederPluginApi
      org/pkl/config/java/mapper/classes/ba.sake.deder.buildinfo.properties
    BuildInfoPlugin.pkl
  scripts/gen-plugin-bindings.sh
  src/ba/sake/deder/
    BuildInfo.java                                  # Generated from .pkl (committed)
    buildinfo/
      BuildInfoPluginImpl.scala                     # Implements DederPluginApi
      BuildInfoConfigNormalizer.scala               # Defaults + overrides merge
      BuildInfoGenerationTasks.scala                # Single source generator task
      GitSupport.scala                              # Runs git rev-parse HEAD
  test/src/ba/sake/deder/buildinfo/
    BuildInfoConfigNormalizerSuite.scala
    BuildInfoGenerationTasksSuite.scala
    GitSupportSuite.scala
  examples/consumer/
```

### Task flow

```
buildInfoGenerate (SourceGenerator)
  → normalize config  (BuildInfoConfigNormalizer)
  → collect values    (module metadata, git, system, extra)
  → generate .scala   (string interpolation)
  → write to ctx.out / "sources" / <packagePath>
  → return ctx.out
```

Single task, no dependencies on other tasks (version task pending).
`TaskKind.SourceGenerator` ensures the output directory is added to compilation source paths.

### Components

- **`BuildInfoPluginImpl`** — Entrypoint. Evaluates Pkl config, returns 1 task.
- **`BuildInfoConfigNormalizer`** — Merges `defaults` + per-module `modules` overrides
  using the same pattern as `ProtobufConfigNormalizer`. Resolves nullable overrides
  with `overrideValue.getOrElse(defaultValue)`. Validates config (no deferred features
  to reject yet).
- **`BuildInfoGenerationTasks`** — Single `CachedTaskBuilder` source generator task.
  Collects all values, renders the Scala source, writes it to `ctx.out / "sources"`.
- **`GitSupport`** — Runs `os.proc("git", "rev-parse", "HEAD").call(cwd = moduleRoot)`
  and returns trimmed output. Returns `"unknown"` on failure (no git repo, not installed).

## Build integration (`deder.pkl`)

Add a second `CreateScalaModules` block alongside the existing protobuf one:

```pkl
local const buildInfoModules =
  new CreateScalaModules {
    root = "deder-build-info"
    id = "deder-build-info"
    layout = "default"
    template = new ScalaModule { /* shared template */ }
    testTemplate = (template.asTest()) { /* shared test template */ }
  }.get
modules { ...pluginModules.all ...buildInfoModules.all }
```

## Test plan

| Suite | Tests |
|-------|-------|
| `BuildInfoConfigNormalizerSuite` | Defaults resolution; module override wins; `objectName` defaults; `includeGitHash`/`includeTimestamp` default `false`; extra values merge; disabled module |
| `BuildInfoGenerationTasksSuite` | All always-on fields present in output; git/timestamp omitted when false; package declaration present/absent; `toMap` contains all active keys; extra values sorted alphabetically |
| `GitSupportSuite` | Returns hash string when git available; returns `"unknown"` when no git repo |

## Open items / future work

- Replace hardcoded `moduleVersion` with `coreTasks.moduleVersionTask` dependency
  once `CoreTasksApi` is expanded upstream.
- Java class output support (currently Scala-only; Java modules consume the
  generated Scala object since it's JVM-compatible).
